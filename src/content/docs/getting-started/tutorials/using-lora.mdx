Lora is a web-based user interface that let's you visualize accounts, transactions, assets and applications on an Algorand network and also provides ability to deploy and call smart contracts. This works for TestNet, MainNet and also LocalNet. While AlgoKit surfaces both a programming interface and a command line interface for interacting with Algorand, it also allows you to quickly open Lora so you can see what's happening visually.

If you wan to learn more about LORA you can read through the documentation here:

<LinkCard title='LORA' href='/algokit/lora/overview' description='LORA Docs' />

If you want to see LORA in action, you can use the hosted version at https://lora.algokit.io/, or if you have [AlgoKit Installed](/getting-started/tutorials/get-ready-to-build#install-algokit), LORA can be launched from the CLI by running the following command:

```shell showLineNumbers=false frame=none
algokit explore
```

By default it will open Lora and point to LocalNet (It will be displayed as `LocalNet` in the upper right hand corner), but you can pass in parameters to point it to TestNet and MainNet too, or change it manually through the interface. 

:::note
If you are using Safari, then it won't work against LocalNet and you will need to open it in a different browser.
:::

## Create / Connect local account for testing

To issue commands against the LocalNet network you need a wallet with ALGO in it. LORA gives you three main options for connecting to a wallet: `Connect KMD`, `Connect MNEMONIC`, and `Connect Lute`:

- `Connect KMD`: LORA provides an unencrypted default wallet to begin with.

- `Connect MNEMONIC`: You can manually input a MNEMONIC for an account you own.

- `Connect Lute`: You can create local accounts from [Lute](https://lute.app/) and connect to them.
 
- You can also generate additional accounts within LORA by selecting the `Create and fund a new LocalNet account` or going to the `Fund` tab on the lefthand navigation. These will be added to the KMD keychain and you can switch between them in the dropdown after connecting the KMD wallet. 

In this guide, we'll domeonstrate using the default KMD wallet.

Select `Connect wallet` located at the top right hand side of the webpage and you will be prompted with the three wallet choices. Choose the `Connect KMD` option. This will prompt you to enter the KMD password. If this is your first time building on Algorand, you do not have a KMD password so leave it blank and click `OK`. This will connect the KMD account to Lora so you can use that account for signing transactions from the Lora user interface.

## Navigating 

In the lefthand navigation you will find 4 tabs:

- `Explore`: The blockchain explorer.
- `App Lab`: Where you can find and manage the apps you create.
- `Txn Wizard`: Build and submit transactions.
- `Fund`: Generate/Fund additional accounts.

### Explore

If you are connected to TestNet or MainNet you will see new blocks and transactions appear as they are added to the blockchain. You can click on any of them to drill down deeper. 

However, if you are connected to the default LocalNet then you will only see new blocks appear when you have submitted a transaction and causes the chain to produce a new block.

<Figure src={lora1} alt='Explorer' />

### App Lab

In the App Lab you can connect to and application you've already deployed by inputting the application ID. 

You can also deploy a new application that you've already compiled to the blockchain through the web interface. We will walk you through this process in a later tutorial. 

<Image src={applab} alt='App Lab' />

### Txn Wizard

The Txn Wizard allows you to prepare transactions and submit them to the blockchain in a visual and interactive way. You can submit one transaction at a time, or you can prepare a series of transactions and take advantage of Algorands native atomic transation group cabability. 

Additionally, you can run the transaction(s) in simulate mode which will inform you if the transaction would be successfully without having to submit it to the chain. 

<Image src={txnwizard} alt='Txn Wizard' />

### Fund

If you are connected to LocalNet you have access to infinite test Algos and can use the Fund tab to send or mint them into any accounts you want. 

However, this tab is disabled if you are connected to MainNet and if you are on TestNet, you will need to go through an additional step of varifying your email in order to get a drip of TestNet Algos. 

<Image src={fund} alt='Fund' />
