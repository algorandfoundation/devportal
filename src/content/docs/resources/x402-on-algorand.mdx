## What is x402

The HTTP error code 402 Payment Required has been in the original specification of the HTTP protocol since the 1990s and is described as being reserved for future use.

In 2025, the Coinbase developer platform has picked up the implementation of this error code and created the x402 protocol, an internet-native emerging payment standard. It enables novel payment methods such as micropayments, pay-per-API-requests, and machine-to-machine payments.

### Roles in x402

There are different roles in a typical X402 scenario.

A merchant wants to provide a resource. They decide on pricing, access rules, and payment terms.

Resource servers protect access to these resources and enforce the x402 requirements.

The facilitator verifies payments and checks settlements.

Clients, such as AI agents, have different ways of requesting an x402 protected resource, e. g. via HTTP, MCP or A2A.

### x402 flow

A typical request flow in x402 looks as follows:

At a protocol level, x402 follows a predictable pattern:

1. A client or agent requests a protected resource.
1. The server responds with HTTP 402 and structured payment requirements.
1. The client submits payment proof.
1. The facilitator verifies settlement.
1. The server returns the requested resource.

### Common x402 use cases

*Pay-per-API access:* Charge per request without managing plans or quotas. 

*Agent-to-service payments:* Allow AI agents to discover services and pay for usage autonomously.

*On-demand compute and inference:* Meter and charge for execution time or inference requests.

*Premium data and content APIs:* Sell access to high-value data without subscriptions.

*Internal service monetization:* Enable usage-based billing and cost attribution across microservices.
