        <a href='12-program-signing/'>Program Signing (Delegated Logic Signatures) with KMD</a>
      </td>
      <td>
        <p>
          This example demonstrates how to sign programs/contracts using the KMD sign_program()
          method. It shows:
        </p>
        <ul>
          <li>Creating a simple TEAL program (logic signature)</li>
          <li>Compiling the TEAL program using algod's teal_compile</li>
          <li>Signing the compiled program bytes with sign_program()</li>
          <li>Understanding the resulting signature for delegated logic signatures</li>
          <li>How to use the signature with a LogicSigAccount</li>
        </ul>
        <p>
          What is Program Signing? Program signing creates a "delegated logic signature" (delegated
          lsig). This allows an account holder to authorize a smart contract (TEAL program) to sign
          transactions on their behalf. When you sign a program: 1. You're attesting that you
          authorize this program to act for your account 2. Transactions signed by this delegated
          lsig will be authorized by your account 3. The program logic determines which transactions
          are approved
        </p>
        <p>Use cases for delegated logic signatures:</p>
        <ul>
          <li>Recurring payments (program checks amount and frequency)</li>
          <li>Subscription services (program validates payment recipients)</li>
          <li>Limited spending authorizations (program enforces constraints)</li>
          <li>Conditional transfers (program checks external conditions)</li>
        </ul>
        <p>Prerequisites:</p>
        <p>Covered operations:</p>
        <ul>
          <li>sign_program() - Sign a compiled TEAL program with a wallet key</li>
        </ul>
      </td>

          <li>Signing the program with the second participant (completing the multisig)</li>
          <li>Understanding delegated multisig logic signatures</li>
        </ul>
        <p>
          What is Multisig Program Signing? Multisig program signing creates a "delegated multisig
          logic signature". This combines two powerful concepts: 1. Multisig: Requiring multiple
          parties to approve 2. Delegated Logic Signatures: Authorizing a program to act on behalf
          of an account
        </p>
        <p>With a delegated multisig lsig:</p>
        <ul>
          <li>The multisig account authorizes a program to sign transactions</li>
          <li>Multiple parties must sign the program (meeting the threshold)</li>
          <li>Once signed, the program can authorize transactions within its logic</li>
          <li>No further interaction needed from the multisig participants</li>
        </ul>
        <p>Use cases for delegated multisig logic signatures:</p>
        <ul>
          <li>Multi-party controlled recurring payments</li>
          <li>Joint account automation (e.g., business partners authorizing limit)</li>
          <li>Escrow with automated release conditions</li>
          <li>DAO treasury with programmatic spending rules</li>
        </ul>
        <p>Prerequisites:</p>
        <p>Covered operations:</p>
        <ul>
          <li>
            sign_multisig_program() - Sign a compiled TEAL program with a multisig participant
          </li>
        </ul>
